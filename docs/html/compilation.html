Compiling and Running an Application

Introduction

To get started working with AXIS Embedded Development SDK, take a look at one of the example applications located in the apps/ directory in the SDK root directory. The analyze application is an example of a Linux daemon that uses several of the interfaces provided by the SDK.

Below, we will show how to create a package for the analyze application and how to upload the package to an Axis product.

  • Creating the application package
  • Installing the package in the Axis product

Creating the application package

Start by sourcing the file init_env in the SDK root directory (names might differ depending on SDK version):

user@host$ pwd && ls
/home/user/axis/emb-app-sdk_2_0
apps  docs  init_env  libs  licenses.txt  RELEASENOTES  target  tools
user@host$ source init_env

Sourcing init_env will:

  • Check compiler location and revision
  • Set the AXIS_TOP_DIR environment variable
  • Append AXIS_TOP_DIR/tools/build/bin to your path
  • Append AXIS_TOP_DIR/tools/scripts to your path
  • Define environment variables for the debugger

This step is required to be able to use the SDK tools and must be repeated each time you open a new terminal/console.

Change directory to apps/analyze: 

$ cd apps/analyze

The apps/analyze directory contains the following files: 

$
Makefile                    The application Makefile.
analyze.h                   Header file.
analyze_error.c             Source file
analyze_error.h             Header file.
analyze_util.c              Source file.
analyze_util.h              Header file.
declarations/               Directory containing event declaration.
http_cgi_paths.txt          Configuration file containing CGI-paths.
main.c                      Source file.
motion_detection.c          Source file.
motion_detection.h          Header file.
package.conf                The application package configuration.
param.conf                  The application parameters definition.
storage.c                   Source file.
storage.h                   Header file.

To compile the application and create an eap package file, use: 

$ create-package.sh

This command will:

  • Create the file .target-makefrag
  • Execute make in the current directory
  • Check that the file package.conf exists and does not contain any errors
  • If required, ask for the parameters and create package.conf
  • Create and empty the file param.conf
  • Create an eap package including the application binary, html files (if any) and configuration files.

The command will create one package for each CPU (processor) supported by the current version of the SDK. To create a package for one specific CPU only, use: 

$ create-package.sh <CPU>

For example: 

$ create-package.sh artpec-4

Or, 

$ create-package.sh mipsisa32r2el

To see the new files, list the contents of the directory. In the example below, only the new files are shown. 

$
analyze                                 The application executable (binary file).
analyze_2_0_mipsisa32r2el.eap           The package file ready-to-upload for MIPS architecure (artpec-4).
motion_detection.o                      ELF object-file.
main.o                                  ELF object-file.
analyze_util.o                          ELF object-file.
analyze_error.o                         ELF object-file.
storage.o                               ELF object-file.
.target-makefrag                        The .target-makefrag specifies the current buildtype.

Important: make requires the hidden file target-makefrag to work. If using make directly instead of create-package.sh, the script must still be run to generate target-makefrag. Also note that make will not update the eap package. The package must be updated using create-package.sh.

Installing the package in the Axis product

The eap package analyze_2_0_mipsisa32r2el.eap in the apps/analyze directory can now be uploaded to the Axis network video product.

Use eap-install.sh to upload and install the package. The IP address or host name of the Axis product and the root password must be supplied. The script can be used to install, start, stop and remove the package. To get help, call the script without any options: 

$ eap-install.sh
Use      eap-install.sh to install a package on the target
Usage:   eap-install.sh <target_ip>  <password> <action>
Example: eap-install.sh 192.168.0.90 mypasswd [install|start|stop|remove]
Example: eap-install.sh 192.168.0.90 mypasswd install

The script remembers the target_ip and password after the first
successful execution. After this you can simply write:
Usage:   eap-install.sh <action>
Example: eap-install.sh install

Note: The curl library must be installed for eap-install.sh to work. See section Prerequisites.

Application packages can also be uploaded and installed using the Axis product's web pages. Go to page Setup > Applications. The page can also be accessed using http://camera_ip/devtools.shtml
