Example Applications

Example applications

The example applications are located in the apps/ directory in the SDK root diretory. To compile and upload an example appliction, run: 

user@host$ cd apps/<application_name>
user@host$ make clean
user@host$ create-package.sh <cpu>
user@host$ install-package.sh <camera_ip> <rootpwd> install

The option <cpu> can be set to artpec-4 (mipsisa32r2el) or ambrella (armv6). If <cpu> is omitted, all packages are created. If more than one packages is created, install-package.sh will prompt for the architecture of the Axis product.

Example: 

user@host$ cd apps/analyze
user@host$ make clean
user@host$ create-package.sh mipsisa32r2el
user@host$ install-package.sh 192.168.0.90 my_passwd install

Included examples

The apps directory under the SDK root directory contains a collection of example applications.

  • axevent - The event interface example applications
  • axhttp - The http interface example application
  • licensekey - The license key interface example application
  • axparameter - The parameter interface example application
  • axserialport - A serial port interface example application
  • axptz - The Pan-Tilt-Zoom (PTZ) interface example application
  • axstorage - The storage interface example application
  • axaudio - An audio interface example application
  • hello_glib - A less advanced 'Hello world' application
  • analyze - A video analytics example application
  • vidcap - The video capture interface example application

axevent - The event interface example applications

The event interface makes it is possible to add an event to the Axis product's built in event handling system
as well as subscribe to system events. The axevent interface was introuduced in SDK version 2.0.
The example ax_event_subscription_example.c shows how to subscribe to an event.
ax_event_key_value_set_example1.c and ax_event_key_value_set_example2.c shows how to use and set a key value.
ax_event_property_state_declaration_example.c shows how to declare a property state event.
ax_event_stateless_declaration_example.c shows how to set up a stateless event.

axhttp - The http interface example application

The axhttp application uses the axhttp interface introuduced in SDK version 2.0.
This interface is used to create a CGI interface to the application.
See ax_http_serve_request.c for more information.

licensekey - The license key interface example application

This application shows how to use the license key interface. See package.conf for more information.

axparameter - The parameter interface example application

The axparmeter application axparameterexample.c uses the axparameter interface
introuduced in SDK version 2.0.

axserialport - A serial port interface example application

axserialport_example.c uses the serial port interface which can be used to communicate with external hardware.
Note that only selected Axis products are equipped with an external serial port.

axptz - The Pan-Tilt-Zoom (PTZ) interface example application

axptzexample.c uses the PTZ interface available on selected Axis products.
The axptz interface was introuduced in SDK version 2.0.

axstorage - The storage interface example application

The axtorage application ax_storage_example.c uses the storage interface available on selected Axis products.
The axstorage interface was introuduced in SDK version 2.0.

axaudio - An audio interface example application

axaudioexample.c is an example application that uses the audio interface introuduced in SDK version 2.0.
The audio interface can be used to capture and analyse audio data on selected Axis products.

hello_glib - A less advanced 'Hello world' application

The hello_glib example application that utilizes the axhttp interface and basic glib functionality to make a simple 'hello world' application.
When started, the hello world message is displayed when the CGI, setup by the application, is accessed.

analyze - A video analytics example application

The analyze application uses the following interfaces: axevent, axhttp, axparameter, axstorage and libvidcap - the video capture interface.
The analyze application can be configured and enabled/disabled from the http Settings page. When enabled, the application will set up a CGI which
can used to monitor motion event(s) and the level of motion. The CGI has options for configuring, creating and deleting motion detection windows.
The application will also send out a system event when motion is detected using the axevent interface.

vidcap - The video capture interface example application

vidcap.c uses the video capture interface.
