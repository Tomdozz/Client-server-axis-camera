Detailed Description

Event Interface.

Deprecated:
This API has been deprecated and replaced by the AXEvent API.

The Event Interface contains functions that interact with the Axis product's built in event handling feature. Events that are added using the event interface will be integrated with the Axis event handling and configured using the product's built in web interface or Axis VAPIX interface. The event interface contains functions for adding, removing and triggering events. The associated trigger name that will be shown in the Triggered By... list on the product's Triggered Event Type Setup web page.

Axis Event Handling Feature

An event is a set of rules describing how and when the Axis network video product should perform certain actions. For example, the product can upload images to an FTP server when motion is detected. More information about the Axis event handling feature can be found in the Axis VAPIX documentation and in the product's online help.

Example

The following example adds an event to the Axis product's built in event handling feature with the associated name "Trigger". It will then trigger (activate and deactivate) the event every 10 seconds until the program is terminated and the event is removed.

#include <unistd.h>
#include <signal.h>
#include <event.h>
static volatile int exit_signal = 0;
static void
handle_sigterm(int signo)
{
exit_signal = signo;
}
int main()
{
event_handler_type *event_handler;
event_type *event;
struct sigaction sa;
sa.sa_flags = 0;
sigemptyset(&sa.sa_mask);
sa.sa_handler = handle_sigterm;
sigaction(SIGTERM, &sa, NULL);
sigaction(SIGINT, &sa, NULL);
event_handler = event_handler_init("eventapp", EVENT_HANDLER_SEND);
event = event_add(event_handler, "Trigger", NULL);
while(!exit_signal) {
event_send(event, EVENT_ACTIVATE_DEACTIVATE, NULL);
sleep(10);
}
event_remove(event);
event_handler_cleanup(event_handler);
return 0;
}

int acap_event_subscribe ( event_handler_type *  event_handler,
void *  user_data,
EventSubCallback  app_callback,
  ... 
)

Subscribes to a set of parameters defined in the cameras event-system.

Parameters
event_handlerPointer to the structure associated with the event handler.
user_dataCustom data that will be returned in the app_callback. The caller is responsible for freeing this memory.
app_callbackThe callback function that will be used to notify the application when there has been an update to parameters that has been subscribed to.
...A list that shall contain and arbitrary number of the full set (key, value, value_type) building up a subscribe expression. The key and value are both strings and the value_type shall be specified according to the enum ValType. When a certain value has been reached for a certain kye specified in the subscribe expression a notification will be sent to the application through the callback-function. If all changes to a certain parameter is wanted the wild-card "*" shall be used as value and the value_type shall be VAL_TYPE_STRING. The list shall end with NULL.
Returns
0 on success, -1 on failure.
